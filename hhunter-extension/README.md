# HHunter — расширение Chrome (Manifest V3)

Отклики на [hh.ru](https://hh.ru) в браузере: текст вакансии уходит на бэкенд HHunter, письмо генерируется (Gemini), вставляется в форму hh.ru и сохраняется в вашей БД через API.

## Требования

- Установленный бэкенд HHunter (FastAPI) и учётная запись на веб-сайте проекта.
- В настройках сайта: ключ Gemini и текст резюме; в разделе «Поиск» — сохранённые параметры (лимит и паузы).

## Установка

1. Откройте `chrome://extensions`, включите **Режим разработчика**.
2. **Загрузить распакованное расширение** → выберите эту папку `hhunter-extension`.
3. В popup расширения при необходимости откройте «Настроить URL API»:
   - **URL API** — тот же хост, что у `VITE_API_BASE` фронта (например `http://localhost:8000`).
   - **URL сайта HHunter** — origin фронта (например `http://localhost:5173`), откуда читается `localStorage` ключ `hhunter_token`.
4. В `manifest.json` в `host_permissions` должны быть ваши реальные origin фронта и API (для production добавьте строки с `https://…`).

## Использование

1. Войдите на сайт HHunter в Chrome.
2. В popup нажмите **Прочитать токен** (должна быть открыта вкладка с сайтом).
3. Откройте вкладку **hh.ru** — поиск вакансий или одна вакансия.
4. Нажмите **Запустить**. Расширение по очереди нажимает «Откликнуться», вставляет письмо и сохраняет результат на сервере.

Капча: расширение останавливается; решите капчу вручную и запустите снова.

## Структура

- `manifest.json` — разрешения и точки входа.
- `background.js` — service worker, запросы к `/extension/*`.
- `content.js` — логика на страницах hh.ru (селекторы могут потребовать обновления при смене вёрстки).
- `popup/` — интерфейс и чтение токена через `chrome.scripting`.
